# 자연어 AI 디스코드 봇 컨셉 문서

## 한 줄 컨셉

명령어를 외우지 않아도, 사용자가 평소 말하듯 요청하면 필요한 기능을 실행하는 관리자 DM 중심의 개인 비서형 디스코드 봇.

## 문제 정의

기존 명령어형 봇은 사용자가 정확한 명령어와 인자 순서를 기억해야 한다. 기능이 늘어날수록 `!날씨 설정 서울`, `!주간날씨 부산`, `/관리자 로그 10` 같은 형식을 외우기 어렵고, 간단한 요청도 사용법 확인이 먼저 필요해진다.

이 프로젝트의 방향은 명령어를 없애는 것이 아니라, 명령어 위에 자연어 인터페이스를 얹는 것이다. 사용자는 자연어로 요청하고, 봇은 의도를 분류해 기존 기능을 실행한다.

## 제품 성격

이 봇은 공개 서버용 다목적 챗봇이 아니라 관리자 개인 운영 도우미에 가깝다.

- 주 사용 환경은 `ADMIN_ID` 사용자의 DM이다.
- 서버 채널 대화보다 개인 요청 처리에 초점을 둔다.
- AI는 판단과 답변 생성을 담당하고, 실제 기능은 검증된 기존 코드가 실행한다.
- 기능 확장보다 안전한 실행, 명확한 fallback, 운영 가능성을 우선한다.

## 핵심 가치

### 1. 자연어 우선

사용자는 정확한 명령어를 몰라도 된다.

```text
서울 날씨 알려줘
이번 주 부산 날씨 어때?
오늘 운세 봐줘
긱뉴스 번역해줘
최근 로그 10개 보여줘
```

### 2. 한글 명령어 유지

자연어가 실패하거나 자동화가 필요할 때는 한글 명령어를 직접 사용할 수 있다.

```text
!날씨 서울
!주간날씨 부산
!운세
!긱뉴스
/관리자 로그 10
```

현재 명령어 체계는 한글 기준이다. `핑`과 `info` 기능은 제거된 상태다.

### 3. AI는 실행자가 아니라 라우터

AI가 모든 작업을 직접 처리하지 않는다. AI의 핵심 역할은 사용자의 문장을 실행 가능한 intent와 인자로 바꾸는 것이다.

AI가 하는 일:

- 자연어 의도 분류
- 지역, 개수, 공지 내용 같은 인자 추출
- 애매한 요청 감지
- 일반 질문 답변
- 검색이 필요한 질문 요약

기존 코드가 하는 일:

- 기상청 날씨 API 조회
- 사용자별 날씨 지역/알림 설정 저장
- 운세 생성 및 하루 1회 캐싱
- 긱뉴스 조회, 번역, 이력 저장
- 관리자 로그/데이터 조회
- 공지 발송
- 자체 상태 점검

### 4. 위험한 요청은 확인

아래 요청은 자연어로 명확하게 들어와도 바로 실행하지 않는다.

- 공지 발송
- 데이터 초기화
- 날씨 알림 설정 변경
- 기본 날씨 지역 삭제

봇은 사용자의 요청을 요약하고 `확인` 또는 `취소`를 기다린다.

### 5. 실패는 짧고 실행 가능하게 안내

실패 메시지는 내부 오류를 그대로 노출하지 않는다. 사용자가 다음에 무엇을 입력하면 되는지 짧게 안내한다.

```text
요청을 정확히 이해하지 못했습니다.
예: "서울 날씨 알려줘", "오늘 운세 봐줘", "긱뉴스 번역해줘"
```

날씨 지역 fallback처럼 봇이 추정해서 처리한 경우에는 사유를 함께 말한다.

## 현재 기능 범위

### 일반 기능

- 오늘 날씨 조회
- 주간 날씨 조회
- 기본 날씨 지역 설정/해제
- 날씨 DM 알림 설정/해제
- 오늘의 운세 조회
- 긱뉴스 기사 번역
- 게임 센터 링크
- 내 정보 조회
- 도움말 조회

### 관리자 기능

- 관리자 명령어 목록 조회
- 최근 명령어 로그 조회
- 저장 데이터 현황 확인
- 공지 발송
- 뉴스 콘텐츠 테스트
- 운세 데이터 초기화
- AI 검색 답변
- 봇 상태 점검

## 대표 시나리오

### 날씨 조회

사용자:

```text
오늘 서울 날씨 어때?
```

봇 처리:

- intent: `weather.today`
- 실행: `날씨` 명령
- 결과: 현재 날씨, 최저/최고 기온, 강수확률 출력

### 주간 날씨

사용자:

```text
이번 주 부산 날씨 알려줘
```

봇 처리:

- intent: `weather.weekly`
- 실행: `주간날씨` 명령
- 결과: 내일부터 7일간 예보 출력

### 운세

사용자:

```text
오늘 운세 좀 봐줘
```

봇 처리:

- intent: `fortune.today`
- 실행: `운세` 명령
- 결과: 오늘 운세 출력
- 같은 날 다시 요청하면 저장된 운세 반환

### 긱뉴스

사용자:

```text
긱뉴스에서 볼만한 거 하나 번역해줘
```

봇 처리:

- intent: `geekNews.translate`
- 실행: `긱뉴스` 명령
- 결과: GeekNews 상단 기사 본문을 한국어로 번역해 출력

### 관리자 로그

사용자:

```text
최근 로그 10개 보여줘
```

봇 처리:

- intent: `admin.log`
- 조건: 관리자 DM
- 실행: 관리자 `로그` 명령

### 공지 발송

사용자:

```text
공지로 오늘 밤 11시에 점검한다고 보내줘
```

봇 응답:

```text
공지 발송 요청으로 이해했습니다.
내용: 오늘 밤 11시에 점검합니다.

실행하려면 "확인", 취소하려면 "취소"라고 답해주세요.
```

## 운영 원칙

- 배포는 Termux 서버의 PM2 프로세스 기준으로 진행한다.
- 로컬 Mac 스케줄러를 사용하지 않는다.
- `dist/index.js`를 서버에 반영하고 `pm2 restart discord-bot --update-env`로 재시작한다.
- 운영 환경의 `.env`는 저장소에 커밋하지 않는다.

## 성공 기준

아래 요청을 명령어 없이 처리할 수 있으면 현재 자연어 UX 목표를 만족한다.

- `서울 날씨 알려줘`
- `이번 주 서울 날씨 어때?`
- `내 기본 지역 서울로 설정해줘`
- `날씨 알림 켜줘`
- `오늘 운세 봐줘`
- `긱뉴스 하나 번역해줘`
- `최근 로그 10개 보여줘`
- `봇 상태 테스트해줘`
- `이 질문 검색해서 답해줘`

동시에 한글 prefix 명령어와 관리자 명령어가 계속 동작해야 한다.

## 제외 범위

현재 범위에 포함하지 않는 것:

- 공개 서버 전체 대상 AI 채팅
- 복잡한 장기 메모리
- 서버별 권한 체계
- 외부 캘린더 연동
- 결제/구독 기능
- 관리자 확인 없는 대량 발송
