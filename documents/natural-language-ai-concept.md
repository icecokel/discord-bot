# 자연어 AI 디스코드 봇 컨셉 문서

## 한 줄 컨셉

명령어를 외우지 않아도 DM에서 자연어로 묻고 요청할 수 있고, 서버 채널에서는 기존 `!` 기능을 조용히 제공하는 한국어 디스코드 비서 봇.

## 현재 방향

이 프로젝트는 공개 서버 전체를 대상으로 한 자유 대화 챗봇이 아니다. 기존 디스코드 봇 기능을 안정적으로 유지하면서, DM 안에서는 자연어 요청과 단순 질문 답변을 지원하는 개인 비서형 도구다.

핵심 방향:

- 일반 기능은 관리자 전용이 아니다.
- 관리자 기능만 `ADMIN_ID`와 DM 조건으로 제한한다.
- 서버 채널에서는 `!` prefix 명령어만 처리한다.
- prefix 없는 자연어 요청과 단순 질문은 DM에서만 처리한다.
- Hermes는 코딩 도우미가 아니라 정확한 답변을 찾아주는 한국어 AI 비서다.
- 최신 정보가 필요한 질문에는 Hermes의 `web` toolset을 사용할 수 있다.
- Hermes AI 답변은 사용자+채널 단위 짧은 대화 맥락을 사용하고, 7턴마다 요약으로 압축한다.
- DM의 자연어 AI 답변은 현재 Discord 메시지와 첨부 메타데이터를 Hermes bridge context로 전달한다. 이미지 첨부는 타입과 크기를 제한해 임시 파일로 내려받고, Hermes에는 로컬 참조 경로만 제공한다.

## 문제 정의

기존 명령어형 봇은 사용자가 정확한 명령어와 인자 순서를 기억해야 한다. 기능이 늘어날수록 `!날씨 설정 서울`, `!주간날씨 부산`, `/관리자 로그 10` 같은 형식을 외우기 어렵다.

동시에 서버 채널에서 봇이 모든 대화에 반응하면 채널 경험이 나빠진다. 그래서 이 프로젝트는 다음 두 가지 사용 방식을 분리한다.

- 서버 채널: 명확한 `!` 명령어 중심
- DM: 자연어 요청과 단순 질문 중심

## 제품 성격

이 봇은 기존 기능 실행기와 질문 답변 비서를 결합한다.

- 일반 사용자는 날씨, 운세, 긱뉴스, 게임 링크, 내 정보, 도움말을 사용할 수 있다.
- 관리자는 로그, 데이터, 공지, 초기화, 테스트 같은 운영 기능을 사용할 수 있다.
- AI는 기존 기능을 대체하지 않고, 사용자의 문장을 intent로 분류하거나 일반 질문에 답한다.
- 위험한 작업은 확인 단계를 거친다.
- 운영은 icenux 서버의 PM2와 GitHub self-hosted runner를 기준으로 한다.
- 관리자는 `!헤르메스 상태`, `!헤르메스 켜기`, `!헤르메스 끄기`로 실행 중인 AI 공급자를 토글할 수 있다.

## 핵심 가치

### 1. 일반 기능은 열려 있어야 한다

날씨, 운세, 긱뉴스 같은 이미 만들어진 기능은 관리자만의 도구가 아니다.

```text
!날씨 서울
!운세
!긱뉴스
!도움말
```

이 기능들은 서버 채널과 DM에서 모두 사용할 수 있다.

### 2. 자연어는 DM에서 조용히 처리한다

사용자는 DM에서 명령어를 외우지 않고 요청할 수 있다.

```text
서울 날씨 알려줘
이번 주 부산 날씨 어때?
오늘 운세 봐줘
긱뉴스 번역해줘
수도권에서 팥빙수 맛있는집 찾아줘
```

서버 채널의 prefix 없는 일반 대화는 처리하지 않는다.

### 3. Hermes는 정확한 답변 비서다

Hermes는 코딩 도우미로 동작하지 않는다. 디스코드 사용자 질문에 대해 정확한 답변을 찾아주는 한국어 AI 비서로 동작한다.

Hermes 답변 원칙:

- 간결하고 실용적으로 답한다.
- 사실 확인과 정확성을 우선한다.
- 확실하지 않은 내용은 추측하지 않고 확인 필요를 말한다.
- 토큰, 비밀번호, 개인키 같은 민감정보를 요구하거나 노출하지 않는다.
- 기존 봇 기능으로 처리 가능한 요청은 직접 꾸며 답하지 않고 해당 기능 사용을 유도한다.

현재 운영에서는 `HERMES_TOOLSETS=web`만 허용한다. `terminal`, `file`, `code_execution`, `browser` 같은 toolset은 디스코드 사용자 입력에 열지 않는다.

관리자 `!헤르메스` 토글은 런타임 상태만 변경한다. PM2 재시작 후에는 `.env`의 `AI_PROVIDER` 값으로 다시 초기화된다. `!헤르메스 초기화`는 현재 사용자+채널의 짧은 대화 맥락을 지운다.

### 4. AI는 라우터이자 답변자다

AI가 모든 작업을 직접 처리하지 않는다. 기능 실행은 검증된 기존 코드가 담당한다.

AI가 하는 일:

- 자연어 의도 분류
- 지역, 개수, 공지 내용 같은 인자 추출
- 애매한 요청 감지
- 단순 질문 답변
- 웹 검색이 필요한 질문 답변

기존 코드가 하는 일:

- 기상청 날씨 API 조회
- 사용자별 날씨 지역/알림 설정 저장
- 운세 생성 및 하루 1회 캐싱
- 긱뉴스 조회, 번역, 이력 저장
- 관리자 로그/데이터 조회
- 공지 발송
- 자체 상태 점검

### 5. 위험한 요청은 확인한다

아래 요청은 자연어로 명확하게 들어와도 바로 실행하지 않는다.

- 공지 발송
- 데이터 초기화
- 날씨 알림 설정 변경
- 기본 날씨 지역 삭제

봇은 사용자의 요청을 요약하고 `확인` 또는 `취소`를 기다린다.

### 6. 실패는 짧고 실행 가능하게 안내한다

실패 메시지는 내부 오류를 그대로 노출하지 않는다. 사용자가 다음에 무엇을 입력하면 되는지 짧게 안내한다.

```text
요청을 정확히 이해하지 못했습니다.
예: "서울 날씨 알려줘", "오늘 운세 봐줘", "긱뉴스 번역해줘"
```

날씨 지역 fallback처럼 봇이 추정해서 처리한 경우에는 사유를 함께 말한다.

## 현재 기능 범위

### 일반 기능

- 오늘 날씨 조회
- 주간 날씨 조회
- 기본 날씨 지역 설정/해제
- 날씨 DM 알림 설정/해제
- 오늘의 운세 조회
- 긱뉴스 기사 번역
- 게임 센터 링크
- 내 정보 조회
- 도움말 조회
- DM 단순 질문 답변

### 관리자 기능

- 관리자 명령어 목록 조회
- 최근 명령어 로그 조회
- 저장 데이터 현황 확인
- 공지 발송
- 뉴스 콘텐츠 테스트
- 운세 데이터 초기화
- 관리자 AI 답변
- 봇 상태 점검

## 응답 조건

| 입력 위치 | 입력 형태 | 처리 여부 | 권한 |
| --- | --- | --- | --- |
| 서버 채널 | `!` 일반 명령어 | 처리 | 모든 사용자 |
| 서버 채널 | prefix 없는 자연어 | 무시 | 해당 없음 |
| DM | `!` 일반 명령어 | 처리 | 모든 사용자 |
| DM | prefix 없는 자연어 기능 요청 | 처리 | 모든 사용자 |
| DM | 단순 질문 | 처리 | 모든 사용자 |
| DM | 관리자 명령어 | 처리 | `ADMIN_ID` |

메시지 처리 순서:

1. 봇 메시지는 무시한다.
2. 관리자 DM 명령어를 먼저 처리한다.
3. `!` prefix 일반 명령어를 처리한다.
4. DM 메시지이면 자연어 라우터로 넘긴다.
5. 확인이 필요한 요청은 pending action으로 저장한다.

## 대표 시나리오

### 서버 채널 날씨 조회

사용자:

```text
!날씨 서울
```

봇 처리:

- 일반 사용자도 실행 가능
- 실행: `날씨` 명령
- 결과: 현재 날씨, 최저/최고 기온, 강수확률 출력

### DM 자연어 날씨 조회

사용자:

```text
오늘 서울 날씨 어때?
```

봇 처리:

- intent: `weather.today`
- 실행: `날씨` 명령
- 결과: 현재 날씨, 최저/최고 기온, 강수확률 출력

### DM 단순 질문

사용자:

```text
오늘 발표된 주요 AI 뉴스 요약해줘
```

봇 처리:

- intent: `ai.answer`
- 공급자: Hermes
- toolset: `web`
- 응답 prefix: `[Hermes] `

### 운세

사용자:

```text
오늘 운세 좀 봐줘
```

봇 처리:

- intent: `fortune.today`
- 실행: `운세` 명령
- 결과: 오늘 운세 출력
- 같은 날 다시 요청하면 저장된 운세 반환

### 긱뉴스

사용자:

```text
긱뉴스에서 볼만한 거 하나 번역해줘
```

봇 처리:

- intent: `geekNews.translate`
- 실행: `긱뉴스` 명령
- 결과: GeekNews 상단 기사 본문을 한국어로 번역해 출력

### 관리자 로그

사용자:

```text
최근 로그 10개 보여줘
```

봇 처리:

- intent: `admin.log`
- 조건: `ADMIN_ID` 사용자의 DM
- 실행: 관리자 `로그` 명령

### 공지 발송

사용자:

```text
공지로 오늘 밤 11시에 점검한다고 보내줘
```

봇 응답:

```text
공지 발송 요청으로 이해했습니다.
내용: 오늘 밤 11시에 점검합니다.

실행하려면 "확인", 취소하려면 "취소"라고 답해주세요.
```

## 운영 원칙

- 운영 서버는 `icenux-ms7b23`이다.
- PM2 프로세스 이름은 `discord-bot`이다.
- GitHub self-hosted runner `discord-bot-icenux`가 배포를 수행한다.
- 배포 workflow는 `main` branch push에서 실행된다.
- runner labels는 `self-hosted`, `Linux`, `X64`, `icenux`, `discord-bot`이다.
- 운영 디렉토리는 `/home/icenux/projects/discord-bot`이다.
- 운영 환경의 `.env`는 저장소에 커밋하지 않는다.
- Hermes의 Discord gateway는 사용하지 않는다. Discord 쓰기/삭제/관리 tool도 Hermes에 제공하지 않는다.

## 성공 기준

아래 요청을 현재 정책대로 처리할 수 있으면 목표를 만족한다.

- 서버 채널에서 `!날씨 서울`
- 서버 채널에서 `!운세`
- DM에서 `서울 날씨 알려줘`
- DM에서 `이번 주 서울 날씨 어때?`
- DM에서 `내 기본 지역 서울로 설정해줘`
- DM에서 `날씨 알림 켜줘`
- DM에서 `오늘 운세 봐줘`
- DM에서 `긱뉴스 하나 번역해줘`
- DM에서 `봇 상태 테스트해줘`
- DM에서 `이 질문 웹에서 찾아서 답해줘`
- 비관리자가 관리자 명령을 실행할 수 없어야 함
- 서버 채널의 prefix 없는 일반 대화에는 응답하지 않아야 함

동시에 한글 prefix 명령어와 관리자 명령어가 계속 동작해야 한다.

## 제외 범위

현재 범위에 포함하지 않는 것:

- 서버 채널 전체 대상 AI 자유 대화
- 서버별 세부 권한 체계
- Hermes terminal/file/code execution toolset 노출
- Hermes Discord gateway 사용
- 복잡한 장기 메모리
- 외부 캘린더 연동
- 결제/구독 기능
- 관리자 확인 없는 대량 발송
