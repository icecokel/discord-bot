# 자연어 AI 디스코드 봇 컨셉 문서

## 한 줄 컨셉

명령어를 외우지 않아도, 사용자가 평소 말하듯 요청하면 필요한 기능을 알아서 실행하는 개인 비서형 디스코드 봇.

## 핵심 방향

현재 봇은 `!날씨 서울`, `!운세`, `!긱뉴스`처럼 사용자가 정해진 명령어를 알아야 한다. 앞으로의 방향은 사용자가 명령어를 기억하지 않아도 자연어로 말하면 봇이 의도를 파악하고 적절한 기능을 실행하는 것이다.

예를 들어 사용자는 아래처럼 말할 수 있다.

- `오늘 서울 날씨 어때?`
- `이번 주 부산 날씨 알려줘`
- `오늘 운세 봐줘`
- `긱뉴스 하나 번역해줘`
- `최근 로그 좀 보여줘`
- `이거 검색해서 정리해줘`

봇은 이 문장을 단순 채팅으로 보지 않고, 실행 가능한 요청인지 판단한다. 실행 가능한 요청이면 기존 기능을 호출하고, 일반 질문이면 AI 답변을 생성한다.

## 제품 성격

이 봇은 공개 서버용 다목적 챗봇보다 개인 DM 비서에 가깝다.

- 명령어 도구가 아니라 대화형 도구
- 기능 나열형 봇이 아니라 요청 해결형 봇
- 일반 채팅 AI가 아니라 실제 기능을 실행하는 AI 에이전트
- 서버 전체 봇보다 관리자 개인 운영 도우미에 가까운 봇

초기 사용 범위는 현재 구조와 동일하게 `ADMIN_ID` 사용자 DM 중심으로 둔다. 이후 안정화되면 일부 기능만 서버 채널이나 허용 사용자에게 확장할 수 있다.

## 사용자 경험 원칙

### 1. 사용자는 명령어를 몰라도 된다

사용자는 `!`, `/admin`, 정확한 키워드를 기억하지 않아도 된다.

기존:

```text
!날씨 설정 서울
!날씨 알림
!주간날씨 부산
```

목표:

```text
내 기본 지역 서울로 해줘
매일 날씨 알려줘
이번 주 부산 날씨 어때?
```

### 2. 봇은 바로 실행할 수 있는 일은 바로 실행한다

명확하고 위험하지 않은 요청은 추가 질문 없이 실행한다.

예:

- `오늘 운세 봐줘`
- `서울 날씨 알려줘`
- `긱뉴스 번역해줘`
- `봇 정보 보여줘`

### 3. 위험하거나 되돌리기 어려운 일은 확인한다

대량 발송, 데이터 삭제, 설정 변경은 확인 없이 실행하지 않는다.

예:

- 공지 발송
- 데이터 초기화
- 알림 설정 변경
- 기본 지역 삭제

봇은 실행 전 사용자의 의도를 요약하고 `확인` 또는 `취소`를 기다린다.

### 4. 애매하면 추측보다 질문한다

지역, 대상, 개수, 내용처럼 실행에 필요한 정보가 부족하면 봇은 한 번만 짧게 물어본다.

예:

```text
어느 지역 날씨를 볼까요?
예: "서울 날씨 알려줘"
```

### 5. 기존 명령어는 계속 지원한다

자연어 전환 후에도 기존 `!날씨`, `!운세`, `/admin test`는 유지한다. 이는 디버깅, 테스트, 자동화, fallback에 필요하다.

## AI의 역할

AI는 모든 일을 직접 처리하지 않는다. AI의 핵심 역할은 사용자의 문장을 실행 가능한 의도로 바꾸는 것이다.

### AI가 하는 일

- 자연어 의도 분류
- 필요한 인자 추출
- 애매한 요청 감지
- 일반 질문에 대한 답변 생성
- 검색이 필요한 질문에 대한 요약 답변

### 기존 코드가 하는 일

- 날씨 API 조회
- 운세 생성 및 캐싱
- 긱뉴스 조회/번역/이력 저장
- 관리자 로그 조회
- 데이터 초기화
- 공지 발송
- 테스트 실행

즉, AI는 판단하고 기존 기능은 실행한다.

## 대표 시나리오

### 날씨 조회

사용자:

```text
오늘 서울 날씨 어때?
```

봇 판단:

- intent: `weather.today`
- region: `서울`

봇 실행:

- 기존 `weather` 명령 실행

### 기본 지역 설정

사용자:

```text
내 기본 지역 판교로 설정해줘
```

봇 판단:

- intent: `weather.setRegion`
- region: `판교`

봇 실행:

- 지역 유효성 확인
- 기본 지역 저장

### 운세

사용자:

```text
오늘 운세 좀 봐줘
```

봇 판단:

- intent: `fortune.today`

봇 실행:

- 기존 운세 기능 실행
- 오늘 이미 생성된 운세가 있으면 같은 결과 반환

### 긱뉴스

사용자:

```text
긱뉴스에서 볼만한 거 하나 번역해줘
```

봇 판단:

- intent: `geekNews.translate`

봇 실행:

- 긱뉴스 상단 기사 조회
- 이미 보낸 기사 제외
- 본문 번역 후 임베드 출력

### 관리자 로그

사용자:

```text
최근 로그 10개 보여줘
```

봇 판단:

- intent: `admin.log`
- count: `10`

봇 실행:

- 관리자 권한 확인
- 로그 조회

### 공지 발송

사용자:

```text
공지로 오늘 밤 11시에 점검한다고 보내줘
```

봇 판단:

- intent: `admin.notice`
- content: `오늘 밤 11시에 점검합니다.`
- requiresConfirmation: `true`

봇 응답:

```text
공지 발송 요청으로 이해했습니다.
내용: 오늘 밤 11시에 점검합니다.
대상: 등록된 유저 전체

실행하려면 "확인", 취소하려면 "취소"라고 답해주세요.
```

## 대화 톤

봇의 말투는 짧고 실용적으로 유지한다.

- 과한 캐릭터성은 넣지 않는다.
- 기능 설명을 길게 늘어놓지 않는다.
- 실행 결과를 먼저 보여준다.
- 실패 시 원인과 다음 행동을 짧게 안내한다.

예:

```text
서울 오늘 날씨입니다.
현재 맑음 12°C, 강수확률 10%입니다.
```

실패 예:

```text
지역을 찾지 못했습니다.
예: "서울 날씨 알려줘", "부산 해운대구 날씨 알려줘"
```

## 기능 범위

### 1차 범위

- 자연어 날씨 조회
- 자연어 주간 날씨 조회
- 자연어 기본 지역 설정/해제
- 자연어 날씨 알림 설정/해제
- 자연어 운세 조회
- 자연어 긱뉴스 번역
- 자연어 봇 정보/help/whoami
- 자연어 관리자 로그/데이터/테스트
- 일반 AI 질문 답변

### 2차 범위

- 대화 문맥 유지
- “거기”, “아까 말한 지역” 같은 참조 처리
- 사용자별 선호 말투
- 반복 요청 자동화
- 스케줄 등록 기능
- 일부 기능의 서버 채널 허용

### 제외 범위

초기 버전에서는 아래 기능을 다루지 않는다.

- 여러 사용자 대상 공개 AI 채팅
- 복잡한 장기 메모리
- 서버별 권한 체계
- 외부 캘린더 연동
- 결제/구독 기능
- 관리자 승인 없는 대량 발송

## 성공 기준

사용자가 명령어를 모르는 상태에서도 아래 요청을 성공적으로 처리할 수 있으면 1차 목표를 달성한 것으로 본다.

- `서울 날씨 알려줘`
- `이번 주 서울 날씨 어때?`
- `내 기본 지역 서울로 설정해줘`
- `오늘 운세 봐줘`
- `긱뉴스 하나 번역해줘`
- `최근 로그 10개 보여줘`
- `봇 상태 테스트해줘`
- `이 질문 검색해서 답해줘`

또한 기존 명령어가 그대로 동작해야 한다.

## 핵심 차별점

이 프로젝트의 핵심은 AI 챗봇을 새로 만드는 것이 아니라, 이미 구현된 실용 기능들을 자연어 인터페이스로 감싸는 것이다.

따라서 구현의 중심은 거대한 대화 모델이 아니라 아래 세 가지다.

1. 정확한 의도 분류
2. 안전한 실행 제어
3. 기존 기능 재사용

이 방향을 지키면 기능이 늘어나도 사용자는 명령어를 외울 필요가 없고, 코드는 기존 명령어 단위로 유지보수할 수 있다.
