# AI 활용 지침

이 문서는 디스코드 봇에서 AI API를 사용할 때의 구현 기준을 정리한다.

## 기본 원칙

- 기능 판별과 인자 추출은 키워드 기반 로컬 규칙으로 처리한다.
- AI는 키워드로 처리되지 않은 일반 질문 답변에 사용한다.
- 날씨, 운세, 긱뉴스, 관리자 기능은 기존 커맨드 실행 로직을 재사용한다.
- 위험한 요청은 키워드로 분류됐더라도 바로 실행하지 않고 확인 단계를 거친다.
- AI 실패가 봇 장애로 이어지지 않도록 항상 안전한 fallback 응답을 준비한다.

## 응답 처리

LLM 응답은 완벽한 구조를 보장하지 않는다. 모델이 JSON 형태를 요청받아도 마크다운 코드 블록, 누락된 쉼표, 설명 문장 등이 섞일 수 있다.

권장 방식:

- 먼저 로컬 키워드 기반 분류로 처리 가능한 요청을 처리한다.
- 키워드로 확정되지 않은 요청은 기능 실행을 시도하지 않고 일반 AI 답변으로 fallback한다.
- 구조화가 필요한 AI 응답도 검증 가능한 최소 필드만 신뢰한다.
- 사용자에게는 내부 파싱 오류 대신 "요청을 정확히 이해하지 못했습니다"처럼 실행 가능한 안내를 제공한다.

## 프롬프트 작성

원하는 출력이 있을 때는 명확한 템플릿을 제시한다.

- 섹션 구분에는 `###`, `---` 같은 명확한 구분자를 쓴다.
- 가능한 경우 원하는 응답 예시를 함께 제공한다.
- "다음 형식으로만 답변"처럼 제약을 분명히 쓴다.
- 고유명사, 제품명, 코드, 명령어는 필요하면 원문 유지 지침을 추가한다.

## 자연어 명령 처리 기준

현재 봇은 한국어 명령어와 자연어 요청을 함께 지원한다.

- 일반 명령어: `!날씨`, `!주간날씨`, `!운세`, `!긱뉴스`, `!게임`, `!내정보`
- 관리자 명령어: `/관리자 로그`, `/관리자 공지`, `/관리자 뉴스`, `/관리자 데이터`, `/관리자 초기화`, `/관리자 테스트`, `/질문`
- 제거된 명령어: `핑`, `info`, `도움말`

제거된 기능을 문서나 프롬프트에 다시 추가하지 않는다.
