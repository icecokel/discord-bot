# AI 사용 지침

## 현재 AI 사용 범위

AI는 일반 사용자 기능 판별에 사용하지 않는다. 현재 AI 사용 범위는 관리자 기능으로 제한한다.

- 관리자 DM의 prefix 없는 메시지는 운영 기본값에서 Hermes session으로 처리한다.
- `!헤르메스 끄기` 상태에서는 primary AI 공급자가 Gemini로 바뀌며, prefix 없는 관리자 DM도 Hermes session/toolset을 사용하지 않는다.
- `/질문 <질문>` 관리자 명령어에서 현재 AI 공급자 기반 단발 답변을 생성한다. 이 경로는 Hermes session 기억과 백그라운드 후속 보고를 사용하지 않는다.
- 긱뉴스 스케줄러가 기사 본문 번역/요약에 AI를 사용할 수 있다.
- 긱뉴스 AI 요약/번역은 Hermes만 사용하며 Gemini fallback을 쓰지 않는다. 실패하면 대체 번역을 보내지 않고 실패 사유를 노출한다.
- 운세 일반 명령어는 제거됐지만, 기존 데이터 조회/초기화를 위한 데이터 파일은 유지한다.

## Hermes 관리자 원칙

Hermes는 관리자 DM에서 서버 작업, 리서치, 브라우저 확인, 파일 read-only 조회를 돕는 작업 에이전트다.

- 사실 확인과 정확성을 우선한다.
- 작업 전후로 확인한 내용과 변경 사항을 간결하게 보고한다.
- 애매한 요청은 추측해서 실행하지 않고 사용자에게 질문한다.
- 토큰, 비밀번호, 개인키 같은 민감정보를 출력하지 않는다.
- Discord 쓰기, 삭제, 채널 관리 도구는 제공하지 않는다.

## 위험 작업 기준

아래 작업은 바로 실행하지 않는다.

- 삭제
- 초기화
- 덮어쓰기
- 강제 재설정
- 권한 변경
- 대량 발송
- 서비스 중단
- 운영 데이터 변경

Hermes는 대상, 영향 범위, 되돌리는 방법을 요약한 뒤 사용자 확인을 받아야 한다.

## 일반 사용자 경로

일반 사용자 DM 자연어 답변과 일반 사용자 명령어는 제거됐다. AI는 일반 사용자 요청을 처리하지 않는다. 일반 사용자가 관리자 명령어를 시도하면 무시가 아니라 권한 없음 응답을 받을 수 있다.
